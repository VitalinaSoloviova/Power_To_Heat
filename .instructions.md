Use a neutral UI base combined with a structured gradient system.

Color tokens:

Neutrals:
background: #F5F5F4
surface: #FFFFFF
border: #E7E5E4
textPrimary: #1C1917
textSecondary: #78716C

Warm gradient (energy, activity):
gradientWarmStart: #FF7A3D
gradientWarmMid: #FF9A5A
gradientWarmEnd: #FFD6A5

Cool gradient (depth, system, storage):
gradientCoolStart: #1E3A8A
gradientCoolMid: #3B82F6
gradientCoolEnd: #7DD3FC

Accents:
primary: #F97316
primarySoft: #FED7AA
secondaryAccent: #6366F1


Design rules:

- Never use raw hex values directly in components. Always use predefined tokens.
- Do not introduce new colors.

- Keep the entire layout neutral (background, cards, borders, text).
- Use gradients only for data visualization and highlights.

- Never use flat colors for charts or visual elements. Always use gradients.

- Warm gradients represent:
  energy, activity, flow, production

- Cool gradients represent:
  depth, storage, system state, inactive layers

- Do not mix warm and cool gradients inside the same component.

- Use orange (warm gradient) as the semantic representation of energy flow.

- Use blue (cool gradient) to represent storage and system depth.

- Keep color distribution balanced:
  ~80% neutral UI
  ~20% gradients

- Avoid strong contrast:
  no pure black (#000000)
  no harsh borders

- Borders must be subtle:
  1px solid border color (#E7E5E4)

- Prefer soft shadows instead of strong borders:
  box-shadow: 0 1px 2px rgba(0,0,0,0.05)

- Cards must follow:
  background: #FFFFFF
  border-radius: 12px–16px
  soft shadow
  no heavy outlines

- Buttons:
  use primary color (#F97316)
  hover = slightly darker tone
  text always white

- Text usage:
  headings and body → textPrimary
  secondary info → textSecondary

- Do not overuse accent colors.
- Gradients must be used only where they communicate meaning (charts, flows, highlights).

- Animations:
  use gradients for flow visualization
  avoid color switching
  prefer opacity and movement over color changes
